# 文档目录 (Docs)

本目录用于存放项目的长期维护文档。与 `plans/` 目录不同，`plans/` 主要用于存放开发过程中的计划、任务拆解和临时性设计草案，而 `docs/` 应存放反映项目当前（或目标）状态的稳定文档。

## 建议存放内容

建议按照以下分类组织文档：

### 1. 架构与设计 (Architecture)
*   **系统架构图**：展示前端、后端、数据库及其他服务的交互。
*   **数据库设计**：ER 图、关键表结构说明（如果有非代码定义的约束）。
*   **技术选型说明**：为什么选择某些特定的库或框架。

### 2. API 接口文档 (API)
*   **接口定义**：如果未完全使用 Swagger/OpenAPI 自动生成，可在此存放接口规范。
*   **协议说明**：WebSocket 消息格式、自定义通讯协议细节。

### 3. 开发指南 (Guides)
*   **环境搭建 (Setup)**：比根目录 README 更详细的本地开发环境配置步骤。
*   **贡献指南 (Contributing)**：代码规范、Git 提交规范、分支管理策略。
*   **测试指南**：如何运行各类测试、测试覆盖率要求。

### 4. 部署与运维 (Ops)
*   **部署手册**：生产环境部署流程、Docker/K8s 配置说明。
*   **配置详解**：`config` 或 `.env` 各项参数的详细含义。

### 5. 产品与用户手册 (Product)
*   **功能清单**：系统支持的功能列表。
*   **用户操作手册**：面向最终用户的操作指引。

## 目录结构示例

```text
docs/
├── README.md           <-- 本文件
├── architecture/       <-- 架构设计图与文档
├── api/                <-- 接口文档
├── guides/             <-- 开发与贡献指南
└── ops/                <-- 部署与运维文档
```
